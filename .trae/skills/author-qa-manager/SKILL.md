# Author Q&A Manager

这个 Skill 用于将复杂的讨论精简为易于阅读的问答对，并统一添加到文档末尾的“作者问答”部分。

## 使用场景
- **严格触发条件**：只有当用户在对话中明确表示“记录到作者问答”、“收录到 Q&A”、“把这个知识点固化到文档问答环节”等类似指令时，才允许调用此 Skill。
- **禁止主动调用**：对于普通的技术解答或未明确要求归档的讨论，严禁调用此 Skill。

## 规范格式

```markdown
---

## 💡 作者问答 (Author Q&A)

**Q：[简短、精准的问题描述]**

**A：** [精简的回答，建议包含 1-2 句核心结论，辅以 2-3 个关键点的列表]。
**结论**：[一句话总结]。
```

## 执行逻辑
1. **深度对齐**：必须**基于对话中提供的详细解答**进行总结，严禁为了精简而丢失核心论点或有趣的类比。
2. **提取核心**：从对话中提取最具价值的一个或多个问题。
3. **精简且完整**：保留核心逻辑、对比及生动的比喻。
4. **定位插入点**：检查文档末尾是否已有 `## 💡 作者问答` 章节：
   - 如果有，则在现有章节末尾追加新的 Q&A。
   - 如果没有，则在文档最后添加该章节。
5. **统一风格**：确保使用中文，并保持专业的“ pair-programmer” 语气。
